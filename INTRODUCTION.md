# opslabs — 浏览器里练真实的线上故障

## 一句话定位

**面向中文开发者的浏览器真实故障排查训练平台。打开即用,零安装,真环境,自动判题。**

对标 Killercoda / SadServers,切入点更贴近中国开发者的日常:学习、面试、真实线上事故复盘。

---

## 我们解决什么痛点

工作里出过事故的人都明白一件事:看日志、查端口、改配置、定位服务故障这些事,靠看教程学不会,靠背八股也练不出来 —— 它需要**手感**。

但"造一个真环境练手"的门槛又太高:真服务器要租、要配、搞坏了不敢收拾;本地起 Docker 又折腾半天环境。绝大多数人最后就是写代码一直写,到出事才慌。

opslabs 把这件事压到一个浏览器标签页里:每道题是一个真实故障,你看到的是真终端、真 Linux、真源码,点"检查答案"后台跑判题。做完关掉标签页,容器自动回收,下次想再练打开就行。

---

## 3 分钟能玩到什么

1. 打开 opslabs 首页,看到场景目录(按难度 / 分类 / 技术栈)。
2. 点一个场景,左边是任务说明(背景故事 + 目标 + 约束),右边是运行环境。
3. 按提示改配置、重启服务、修代码。
4. 点"检查答案",几秒后看到通关结果或失败提示。
5. 通关后给个分,关掉标签页 —— 容器就被回收了。

倒计时是给你参考,不会强杀环境;不小心超时了可以继续做,只有你主动放弃才会结束。

## 四种真实环境,按题材选

opslabs 不是"一种技术栈打天下",不同题材用不同的浏览器内运行时,体验差异很大:

| 题材 | 运行环境 | 示例场景 |
|---|---|---|
| **Linux / 服务 / 网络 / 配置排障** | 浏览器里的 Docker 容器 + 真终端(ttyd) | `hello-world`:在真实 Linux shell 里完成第一个任务 |
| **纯前端视觉问题** | 浏览器内嵌的可运行 HTML / CSS / JS 页面 | `css-flex-center`:用 flex 把一个元素在容器里居中 |
| **Node 工程类修复** | StackBlitz WebContainer + 内嵌 Monaco 编辑器(和 VS Code 同款) | `webcontainer-node-hello`:修一个 Node HTTP 接口的对象字段名 bug |
| **轻量 Linux 引导** | v86 WebAssembly x86 模拟器 + BusyBox,完全跑在你的浏览器里 | `wasm-linux-hello`:在浏览器本地跑的 Linux 里 touch 一个文件 |

sandbox 模式走后端判题(`docker exec check.sh`),其他三种前端自己跑判题并上报 —— 这个设计让我们能在同一个平台里既做重型排障题,也做 3 分钟小练习,而不必为每种题材分别造轮子。

## 当前开放的场景

**现在能端到端通关的 4 个:**

| 场景 | 类型 | 难度 | 预计用时 |
|---|---|---|---|
| 欢迎来到 opslabs | 引导 (sandbox) | ★ | 3 分钟 |
| CSS Flexbox 水平垂直居中 | 前端 (static) | ★ | 5 分钟 |
| Node.js 入门:修复 greeting handler | 后端 (web-container) | ★ | 5 分钟 |
| wasm Linux 欢迎:touch 一个文件 | 运维 (wasm-linux) | ★ | 3 分钟 |

**规划中、即将补齐的 3 个:**

- **本地 dev server 启动失败**(前端 / sandbox / ★★ / 8 分钟):接手项目跑不起来,排查 Node 版本、依赖、端口、配置。
- **API 总是返回 500**(后端 / sandbox / ★★★ / 12 分钟):用户接口一直 500,看日志、查配置、验数据库连接。
- **Nginx 反代 502 排查**(运维 / sandbox / ★★★ / 12 分钟):Nginx 返回 502,后端明明活着,找出 upstream 问题。

后续每个版本会按 guide / frontend / backend / ops 四类均衡扩充,第一批目标是 5-8 个场景全部上线。

## 谁适合用

- **1-5 年的后端 / 运维 / SRE / DevOps**:补 Linux、服务、网络、配置排障的手感。
- **想补 Linux 和服务排障的前端 / 全栈**:很多前端同学修真实线上环境时还不熟 `ps / ss / lsof / tail` 这套工具。
- **正在准备面试的人**:比纯八股更能检验"真的出事了你会怎么查"。
- **计算机专业高年级 / 刚转行的人**:把课本上的概念落到真命令、真日志。

**现阶段不优先服务**:完全零基础(需要专门的入门引导,后面做)、企业大客户(权限 / 审计 / 团队管理能力还没到位)。

## 为什么不是视频,不是文章

看视频和读文章有两个共同问题:你没动手,所以没肌肉记忆;你看到的是整理后的正确路径,而真实故障里 80% 的时间花在"走偏了又回来"。

opslabs 每道题都把你放进一个"还没通关"的状态,你自己看、自己试、自己定位,判题只看最终状态。完成五道题比看五小时视频扎实得多。

## 数据与隐私

我们只在你身边留很轻的一层:

- **容器自动回收**:idle 30 分钟后销毁,通关后 10 分钟也会回收。关标签页就等于离开。
- **不存密码 / 密钥**:请不要在练习容器里输入真实密码、API key、公司机密,容器不是你的安全保险箱。
- **匿名记录**:在接入登录前,我们只用一个浏览器随机生成的 clientID 来复用你的 attempt,没有任何个人信息。
- **日志采集克制**:服务端日志记录访问和错误,不会记录你终端里完整的输入输出。

## 接下来会做什么

按价值优先级:

1. **补齐更多真实工作场景**:第一批目标 5-8 个,覆盖前端 / 后端 / 运维 / 网络 / 数据库。

4. **场景推荐**:根据你已通关的技能,推荐下一题。
5. **AI 复盘助手**:通关后解释你做过的关键动作,给出可以继续学习的命令 / 材料。
6. **面试题包 / 企业实操测评**:试水付费场景,验证是否值得长期投入。

## 技术栈(给关心实现的工程师)

- **后端**:Go 1.25 + Kratos v2,Docker SDK 直连本机 Daemon 管容器,Redis 做 attempt 状态缓存,GORM 落库留作审计。
- **前端**:Vite + React 18 + TypeScript,Monaco 编辑器,Zustand 状态,React Query 数据层。
- **运行时**:Docker 容器 / StackBlitz WebContainer / v86(WebAssembly x86) / 静态 bundle 四套并行。
- **判题**:sandbox 走 `docker exec`,其他三种前端自执行判题并上报。

MVP 阶段单机部署,Docker 容器预分配端口池 10000-19999,GC 每 60 秒扫描回收 idle 30 分钟以上的会话。
